# Support

MinSpec is currently in foundation-building mode.

Support should be treated as best-effort while the foundation is being established.

---

## Project Support

Use GitHub issues for:

- questions
- documentation clarity suggestions
- package boundary questions
- MinSpec-specific installation issues
- architecture clarification
- reproducible bugs in MinSpec packages
- evidence that an existing claim, package boundary, or workflow is wrong

---

## Incubation Boundary

Support issues and feedback do not create contributor status, source authority, or permission to open unsolicited pull requests.

During incubation, MinSpec is not accepting unsolicited external contributors, code pull requests, documentation pull requests, package submissions, recipe submissions, workflow changes, dependency changes, or AI-generated contribution patches.

See `CONTRIBUTING.md` for the current contribution posture.

---

## Not Symfony Support

MinSpec cannot provide official Symfony support.

For Symfony framework issues, use Symfony documentation, community channels, or official support paths.

MinSpec issues should be limited to MinSpec repositories, packages, documentation, recipes, and tooling.
